CrowdEye Backend Architecture & API Ecosystem

This visual summarizes the CrowdEye backend as the central integration layer connecting the mobile application, web dashboard, owner operations, and the IoT console. The backend coordinates authentication, fleet operations, live tracking, alerts, analytics, device communication, and data persistence through secure REST APIs.

Spring Boot Backend Core

The backend acts as the orchestration hub of CrowdEye, enforcing business logic, validating requests, processing incoming bus data, and distributing trusted information to all connected systems.

API Layer

REST controllers expose secure endpoints for apps, dashboards, and devices.

Service Layer

Handles analytics, alerting, occupancy logic, and operational workflows.

Repository Layer

Persists fleet, device, trip, and alert data to the relational database.

Security Layer

JWT authentication, role-based authorization, and access control policies.

Data & Infrastructure

Database: MySQL for users, buses, trips, alerts, logs, and analytics snapshots.
Hosting: AWS EC2 / GCP VM deployment for scalable backend execution.
Docs: Swagger / OpenAPI for endpoint testing and integration.
Monitoring: Device logs, alert logs, system logs, and health checks.

IoT Console Integration

The onboard IoT console collects sensor and camera data inside buses, performs edge processing, and sends only meaningful operational events and measurements to the backend.
